# v0.1 Milestone: One Real Audited System Recovery Loop

## Goal

Build the smallest version of Runlane that can handle one real or simulated system-level incident across Linux, FreeBSD, and OpenBSD, with audited evidence collection, dependency-aware scheduling, impact-scoped verification, human approval, and a privileged recovery action.

## Acceptance criteria

- A server can enroll a node and issue pull tasks.
- Linux, FreeBSD, and OpenBSD agents can report platform capabilities.
- The agent can collect service status, logs, disk, process, and socket facts through platform-native backends or clearly report unsupported capabilities.
- A runbook from a Git fleet repo can be selected and executed.
- Tasks declare resources they read/write and scheduler respects dependency/conflict constraints.
- A verification planner selects checks based on impact scope instead of running a global full gate.
- A model or deterministic stub analyzer can produce a structured proposal.
- A privileged action requires approval and a signed capability lease.
- The local helper refuses invalid, expired, replayed, or locally disallowed leases.
- Every state transition, lease decision, verification decision, skipped check, and action result writes an audit event.
- A final incident report / cognitive receipt can be generated from the ledger.

## v0.1 non-negotiables

- No arbitrary shell action.
- No direct model-to-command path.
- No systemd-only design.
- No permanent broad sudo/doas.
- No chat-bot-specific business logic.
- No per-node repository requirement.
- No blanket full-gate after every small task.
- No unmodeled concurrent mutation.

## Kill criteria

Stop or redesign if, after v0.1:

- it is not useful on the operator's own machines;
- onboarding FreeBSD/OpenBSD feels bolted on;
- relevance-scoped verification is not explainable;
- concurrency is still all-serial or unsafe;
- approval/audit is decorative rather than enforcing behavior;
- the project drifts into a generic agent dashboard;
- Git intent and runtime state become confused again.
