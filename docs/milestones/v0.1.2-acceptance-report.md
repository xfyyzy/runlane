## Not Run

| Check | Reason |
|---|---|
| Real Telegram Bot API call | Not run. The v0.1.2 acceptance scope required an approval-channel boundary without leaking secrets. `scripts/smoke/telegram-approval-live-simulated.sh` exercised Telegram-shaped commands through the adapter and recorded `secrets: not-read`. A secret-backed live Telegram run needs a separate operator-approved task with token/chat/user id handling. |
| Production service restart | Not run. The real-host dogfood target was the controlled demo unit `runlane-demo-unhealthy.service`, and the helper path was dry-run validated. No production service restart was authorized in #50. |
| Linux aarch64, Windows, Darwin, illumos release artifacts | Not run. v0.1.2 release-artifact scope was Linux `x86_64-unknown-linux-musl`; other targets remain outside this milestone's acceptance claim. |
| Package signing, installer publishing, or package registry upload | Not run. v0.1.2 is a validation milestone, not a distribution release with signed packages. |

## Blocked

No v0.1.2 acceptance item is blocked at report time.

Environment exceptions and local lab facts were recorded outside the project
tree at validation time. They include local VM choices such as NAT networking,
OpenBSD native VM validation, snapshot Rust packages for OpenBSD Rust 1.96.0,
and the Linux VM's rustup-based Rust 1.96.0 setup. They do not create project
requirements.

Durable evidence follow-up: issue
[#61](https://github.com/xfyyzy/runlane/issues/61) replaces these ephemeral
local/VM evidence handles with GitHub Actions artifacts and short
source-controlled summaries. One-host local paths remain historical
validation-run details, not portable release artifacts.
